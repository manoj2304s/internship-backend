# Internship Backend Auth System

This project implements a role-aware authentication backend with registration, OTP verification, JWT auth, refresh tokens, password reset, logout, and protected routes.

## Features

- User registration with `name`, `email`, `phone`, and `password`
- Unique email and phone validation
- Password hashing with `bcrypt`
- OTP verification for registration
- Login with email and password
- Login with phone and OTP using `POST /auth/login-otp`
- JWT access token and refresh token flow
- Refresh token rotation with database persistence
- Forgot/reset password flow with 15-minute reset token expiry
- Logout with refresh token invalidation
- Protected route middleware
- Role-based authorization for `user`, `admin`, and `super_admin`
- OTP expiry and attempt limits
- Rate limiting for auth endpoints
- Helmet secure headers
- CORS configuration
- Centralized error handling middleware

## API Endpoints

## OTP and Email Clarification

Yes, OTP verification can be implemented in code only for development if you cannot afford paid SMS or email platforms.

This project supports two modes:

- Real email mode:
  Configure `EMAIL_USER` and `EMAIL_PASS` and the backend sends verification/reset emails through Nodemailer.
- No-cost development mode:
  If email credentials are not configured, the backend simulates delivery and returns a `preview` in the API response. Phone OTP is also simulated and logged because real SMS providers usually cost money.

That means you can fully demonstrate the flow in code and Postman without paying for Twilio or other OTP vendors. For production, you would replace the simulated path with a real SMS or email provider.

## Request Examples

### Register

```json
POST /auth/register
{
  "name": "Jane Doe",
  "email": "jane@example.com",
  "phone": "9876543210",
  "password": "Password123",
  "role": "user",
  "verificationChannel": "email"
}
```

### Verify OTP

```json
POST /auth/verify-otp
{
  "email": "jane@example.com",
  "otp": "123456"
}
```

### Login

```json
POST /auth/login
{
  "email": "jane@example.com",
  "password": "Password123"
}
```

### Login with OTP

Request OTP:

```json
POST /auth/login-otp
{
  "phone": "9876543210"
}
```

Verify OTP:

```json
POST /auth/login-otp
{
  "phone": "9876543210",
  "otp": "123456"
}
```

### Refresh Token

```json
POST /auth/refresh-token
{
  "refreshToken": "your-refresh-token"
}
```

### Forgot Password

```json
POST /auth/forgot-password
{
  "email": "jane@example.com"
}
```

### Reset Password

```json
POST /auth/reset-password
{
  "token": "reset-token",
  "newPassword": "NewPassword123"
}
```

### Logout

```json
POST /auth/logout
{
  "refreshToken": "your-refresh-token"
}
```

### Me

```http
GET /auth/me
Authorization: Bearer <access-token>
```
